# Web 项目重构 - 测试指南

## 📋 测试前准备

### 1. 确保依赖已安装
```bash
# 在项目根目录
pnpm install
```

### 2. 构建 Server 项目
```bash
cd tools/server
pnpm build
```

### 3. 启动服务

**终端 1 - 启动 Server**
```bash
cd tools/server
pnpm start
```

**终端 2 - 启动 Web**
```bash
cd tools/web
pnpm dev
```

## 🐛 常见问题排查

### 问题 1: Server 启动失败

**症状**: `pnpm start` 报错

**解决方案**:
```bash
# 1. 检查端口是否被占用
netstat -ano | findstr :3000

# 2. 重新构建
pnpm build

# 3. 检查数据库文件
# 确保 tools/server/data 目录存在
```

### 问题 2: Web 页面无法访问

**症状**: 浏览器显示连接失败

**解决方案**:
```bash
# 1. 检查 Server 是否正常运行
curl http://localhost:3000/api/health

# 2. 检查 Web 开发服务器是否启动
# 应该看到 "Local: http://localhost:5173"

# 3. 清除浏览器缓存
# Ctrl + Shift + Delete
```

### 问题 3: 项目导入失败

**症状**: 点击导入后显示错误

**可能原因**:
1. 路径不存在
2. 路径格式错误（Windows 使用反斜杠 `\`）
3. 没有 package.json 文件

**解决方案**:
- 确保路径正确
- 使用绝对路径
- 确保目录包含 package.json

### 问题 4: 日志不更新

**症状**: LogViewer 显示但日志不更新

**可能原因**:
1. 任务已完成
2. 轮询被停止
3. API 请求失败

**解决方案**:
1. 检查浏览器控制台是否有错误
2. 检查 Network 标签页的 API 请求
3. 点击"刷新"按钮手动更新
